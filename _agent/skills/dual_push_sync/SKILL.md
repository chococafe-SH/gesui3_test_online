# 統合プッシュ（GitHub & Notion）Skill

このSkillは、ユーザーから「push」またはそれに類する指示があった際に、開発成果物（コード）と文書（アーティファクト）の両方を、それぞれ適切な外部サービスへ同期するためのガイドラインです。

## 同期プロセス

### 1. GitHub同期 (Code)
- **対象**: `lib/` フォルダを中心としたプロジェクトの全てのソースコード。
- **実行**:
    - `git add .` を実行し、全ての変更を検知します。
    - コミットメッセージには、そのセッションで完了した主要なタスクを要約して記載します。
    - `git push origin [branch_name]` を実行してリモートに反映します。

### 2. Notion同期 (Documents)
- **対象**: `brain/<conversation-id>/` 配下の全ての `.md` ファイル（`task.md`, `implementation_plan.md`, `walkthrough.md` 等）。
- **ルール**:
    - **絶対に要約せず**、全文を転記します。
    - `notion-mcp-server` を使用して、所定の「成果物バックアップ」データベースに新規ページを作成します。
    - 階層構造（見出し、リスト、チェックリスト）をNotionのブロック形式で正しく再現します。

## ワークフローの優先順位
1. **GitHub Push**を先に実行し、コードの安全を確保します。
2. **Notion Backup**を次に実行し、文書の記録を外部化します。
3. 最終的にGitHubのPush結果とNotionページのURLを合わせてユーザーに通知します。

## 注意事項
- コードの変更がない場合でも、ドキュメントの更新があればNotionバックアップは実行してください。
- 逆にドキュメントのみの指示であっても、コード側に未コミットの変更があればGitHub Pushも提案、または同時に実行してください。
